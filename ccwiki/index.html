这是一本写给中文读者的《Claude 生态百科全书》——一整张能检索、耐翻的单页工具书，2026 年 7 月编纂。它不堆术语，只做一件事：把散落在英文 GitHub、社区论坛和目录站里的好东西，替你筛过、译过、按「值不值得上手」排好队。如果你手握 Claude Max 订阅、想把这份算力用到极致，又不想天天啃英文文档，这本书就是为你编的。

动手编这本书之前，得先诚实回答一个问题：这活儿，网上是不是早有人干过了？答案是——有一半。英文世界里，「精选清单」（awesome list，社区约定俗成的资源合集）多如牛毛，目录站也琳琅满目。但它们几乎全部有三个共同短板：是英文的、是碎片化的、是写给程序员的。它们把上千个工具堆在一页里，却不告诉一个非程序员「这东西能帮我做什么、值不值得学」；更没有一处会收录金句、方法论和 vibe coding（凭感觉与 AI 对话、让它替你把想法变成成品）的创意作品。

所以本页的定位很清楚：它不跟这些清单比「全」，而是比「懂你」。下面先把最接近本页的几个「现成轮子」逐一拆开评估——它们是本页的上游矿脉，我们从中采矿、提炼、翻译，再按中文读者的口味重新码放。看完你会明白，为什么它们再好也替代不了这一页。

三分钟看懂 Claude 版图

动手之前，先把「Claude 到底是一堆什么东西」理顺。可以这样记：Claude 有一个模型家族（脑子，分四档聪明与快慢）和几种产品形态（用这颗脑子的不同门口）。你的 Max 订阅，本质是买断了「用最贵那几颗脑子、且几乎管够」的权利。

模型家族按「越往下越聪明、越往上越快越省」排：Haiku（最快最便宜，适合跑量）、Sonnet（日常主力，又快又不笨）、Opus（硬核推理与复杂编程）、以及 2026 年 6 月登顶的新一档 Fable（Mythos 级，当前最强，自带「随时深度思考」与百万字上下文）。产品形态则主要是四个门口：Claude 网页/桌面/手机 App（聊天）、Claude Code（在电脑终端或 IDE 里替你干活的编程智能体）、Claude API（给开发者按量调用）、以及 Agent SDK（自己搭智能体的工具箱）。本书主要围绕前两个门口展开。

本页怎么用

这是一整页、可检索的书，不必从头读到尾。三个操作记住即可：其一，左侧「书脊」里的搜索框会实时过滤全书所有卡片，想找什么直接打关键词（中英文都行）。其二，右上角可切换暗色模式（夜读护眼），页面也会跟随你系统的深浅色。其三，每张卡片右下角的星级＝上手难度，不是质量评分——★☆☆☆☆ 是点开即用，★★★★★ 是要花时间折腾；预算有限的注意力，先投给低星高价值的。

三条贯穿全书的总原则
一、用现成轮子，不重复造。想做一件事之前，先假设「大概率有人做过了」——先来本页或上游目录站搜一圈，找到就直接站在它肩上，找不到再自己动手。这是本书第零章就先做「轮子评估」的原因。
二、让贵模型做设计，便宜模型做执行。把最强的 Opus / Fable 用在规划、拆解、定方案这类「一步错步步错」的环节；把 Haiku / Sonnet 用在按方案批量执行的体力活上。好钢用在刀刃，Max 订阅的价值才吃得干净。
三、一切皆可让 Claude 代劳。本书里凡遇到看不懂的英文 GitHub 页面、报错、术语，别硬扛——把整页内容原样贴给 Claude，让它翻译、讲解、甚至直接替你判断「这东西值不值得用」。你不需要变成程序员，你只需要会指挥一个。

最后交代一句本书的取舍标准，好让你信得过这里的每一条：宁缺毋滥。一个条目要被收进来，至少得占「解决真实问题、给人灵感、公认最优」中的一条，拿不准的一律不收。所有星数、赞数只当信号不当门槛，且都标了量级供你判断分量；所有链接都是编者亲手核过的真实页面，绝不凭记忆拼凑。换句话说，这里的「少」是刻意的——替你把海量信息里 90% 的噪音先滤掉，剩下的才值得你花注意力。

模型家族：2026 现状

Claude 现在是一个「按聪明程度和速度分档」的家族。2026 年 6 月，Anthropic 发布了新一代 Claude 5 系列，家族现有四个公开档位：Fable、Opus、Sonnet、Haiku——外加一个只对受邀客户开放的 Mythos。你不必记全，记住一句话就够：越聪明越贵越慢，越轻越快越便宜，把活按难度分给它们即可。

新手须知：「context window（上下文窗口）」指模型一次能读进多少内容，单位是 token（词元，约等于 0.7 个英文单词或半个汉字）。1M token 约合 55 万英文词——足够塞进一整个中型代码库或几本书。「MTok」= 每百万 token 的价格，是 API 按量计费的单位；订阅用户不直接付这个，但它能帮你判断「谁贵谁便宜」。

限额怎么算？Max 套餐有两条每周限额：一条覆盖所有模型，另一条只管 Sonnet——分池意味着你把 Sonnet 跑爆了，其他模型的桶可能还有水。在此之上还有5 小时滚动会话窗口。每周限额在账户固定的时间点重置（在「设置 > 用量」看下次重置时间），与你何时开始用无关。此外 Anthropic 保留按容量设「月度上限」的权利。所以省额度的核心不是省钱，是别在无谓的任务上烧掉高档模型的周额度。

Max 用户省额度七式。下面每一式都来自官方文档的用量最佳实践与 effort / Claude Code 用量说明，专治「周三就把周额度用光」。

  1. 用 /model 分工，让顶级模型只做设计与终审。Claude Code 里 /model 可切 Sonnet（默认）、Opus、Haiku。官方原话：「用 Opus 规划、用 Sonnet 执行」，机械性小活丢给 Haiku。别让 Opus/Fable 去干改错别字这种事。
  2. 调 effort（努力档位）。这是控制「Claude 愿意烧多少 token」的旋钮，五档：max / xhigh / high（默认） / medium / low。低档会让它少调工具、少思考、直奔结果。简单任务降到 medium 或 low，省下的额度很可观；只有真·前沿难题才值得 max。
  3. 善用「快档 / fast」与轻量思考。聊天和非编码场景用 low effort 换更快更省；编码 agent 才上 xhigh。Claude Code 的 ultracode 模式本质是 xhigh + 多代理授权，很吃额度，非必要别常开。
  4. 装个监控工具，别蒙眼开车。ccusage（见下卡）几秒出日/周/月/会话用量报告；Claude Code Usage Monitor 是实时「油表」还能预测撞线。先看「设置 > 用量」的进度条也行。
  5. 上下文卫生。任务之间用 /clear 清掉无关历史，任务中途用 /compact 压缩；对话历史增长最快、最烧 token。文件用「路径引用」而非整段粘贴，CLAUDE.md 保持精简（它每轮都被前置读入）。
  6. 把便宜模型派成子代理。官方对 low 档的建议就是「适合子代理」。让 Opus 当总指挥，拆活给若干 Haiku/Sonnet 子代理并行跑，既快又省主模型额度。
  7. 避开高峰、批量处理。能异步的活丢给网页版 Claude Code 或 Routines（定时任务）在闲时跑；API 侧的 Batch API 有折扣。把相似请求合并进一条消息，别一句一发。Projects 里的文档被缓存，反复引用不重复扣额度。

Claude Code 完全指南

Claude Code 是把 Claude 请进你电脑「命令行」里的工具——它能读你的文件、动手改文件、跑命令，你用大白话交代任务，它自己去办。别被「写代码工具」的名头劝退：批量重命名照片、把一堆 Excel 合并、按你的模板起草标书，它一样能干。本章把它的每一项机制掰开揉碎，配上非程序员也用得上的场景。

它是什么，怎么装

先纠正一个印象：Claude Code 不是聊天框，而是一个「代理」（agent，能自己规划步骤、调用工具、连续行动直到把事做完的 AI）。你说「把这个文件夹里的发票按月份分好」，它会真的去列目录、判断、移动文件，每一步动手前问你一句「可以吗」。它有三种入口，同一套账号、同一套配置通用：

入口适合谁怎么装
终端 CLI（命令行）想要全功能、愿意打字的人Windows 在 PowerShell 里运行 irm https://claude.ai/install.ps1 | iex；Mac/Linux 运行 curl -fsSL https://claude.ai/install.sh | bash。原生安装会在后台自动更新。
VS Code / JetBrains 插件已经在用这些编辑器的人在扩展市场搜「Claude Code」装上，能在编辑器里直接看改动对比。
网页版 / 桌面 App不想碰命令行、想随时随地用浏览器打开 claude.ai/code 即可用；桌面版可并排跑多个任务、可视化审阅改动。

装好后第一件事：进到任意一个项目文件夹，敲 claude 回车，按提示用你的 Claude 订阅账号登录。装好后对它说的第一句话，建议是最没风险、也最能让它「熟悉现场」的一句——「这个文件夹里都是些什么？」它会先读一圈再回答，你也顺势看清它的工作方式。

安全感来自权限：Claude Code 改任何文件、跑任何命令之前，默认都会停下来问你要不要批准。看不懂它要干什么，就回答「不」。放心大胆试，它不会背着你动手。

下面每张卡片＝这机制干什么用＋一个具体场景。难度星越少越省心（★＝开箱即用，★★★★★＝需要折腾）。前几张是每天都会碰到的基础，越往后越进阶。

CLAUDE.md 记忆文件

项目根目录里一个叫 CLAUDE.md 的纯文本文件，每次开新对话它都会先读一遍——相当于「常驻便签」。场景：你每回都要交代「发票文件名用『年月-金额』格式」，写进 CLAUDE.md 后就不用再重复。敲 /init 能让它自动生成一份初稿。建议控制在 200 行内，越短越听话。

官方文档

记忆内置★★☆☆☆

自动记忆 Auto memory

默认开启，不用你写。Claude 一边干活一边把「值得记住的事」自己记下来——比如你纠正过「用 pnpm 别用 npm」、某个测试要先开 Redis。下次开对话它就记得。场景：你说过一次「我的表格第一行是标题别当数据」，之后它默认照办。用 /memory 可随时查看、编辑、删除它记了啥。

官方文档

记忆内置★☆☆☆☆

权限模式 Permission modes

决定它动手前问不问你。四档常用：default（每次都问）、acceptEdits（自动接受改文件，省得一路点确认）、plan（只读只看不改，用来先摸清情况）、bypassPermissions（全不问，只在隔离环境慎用）。场景：让它批量整理 200 张图，先看它头几步没问题，切到 acceptEdits 让它一路跑完。按 Shift+Tab 循环切换。

官方文档

权限内置★☆☆☆☆

Plan Mode 计划模式

让它「先出方案、你点头了再动手」。在这个模式里它只读文件、只查资料，绝不乱改。场景：要重新整理一整个混乱的文件夹，先在 Plan Mode 里让它给出「我打算按这三类分、把这些归档」的计划，你确认无误再放它执行——避免它一上来就把文件挪乱。按 Shift+Tab 切到 plan。

官方文档

规划内置★☆☆☆☆

上下文管理 /clear · /compact

「上下文」＝这次对话里它记得的所有内容，越攒越多会变慢变糊涂。/clear 彻底清空开新篇（旧对话仍可 /resume 找回）；/compact 把长对话压缩成摘要、腾出空间又保留要点。场景：处理完一份 Excel、要换下一份完全无关的表格前，先 /clear，它就不会被上一份的信息干扰。

官方文档

上下文内置★☆☆☆☆

/rewind 检查点回溯

「后悔药」。它每做一步都留了存档点，改坏了可以一键回到之前的状态——代码和对话都能倒回。场景：让它批量替换文件里的某个词，结果发现替换错了，双击 Esc（输入框为空时）打开回溯菜单，选一个之前的节点还原，就当无事发生。有了它才敢放手让 AI 大改。

官方文档

安全网内置★☆☆☆☆

子代理 subagents

派出「分身」去干脏活累活，它们各自在独立的上下文里工作，只把结论汇报回来，不把一堆中间信息塞爆你的主对话。场景：你要在 100 个文档里找出「哪几份提到了预算超支」，让子代理去翻，主对话只收到一句摘要。内置 Explore（探索）、Plan（规划）等代理，也能自定义。

官方文档

代理内置★★★☆☆

hooks 钩子

「一旦发生某事，就自动执行某命令」的规则，由软件强制执行、不靠 AI 自觉。常见触发点：改文件前（PreToolUse）、改文件后（PostToolUse）、任务完成时（Stop）。场景：设一条「每次它改完文件就自动跑一遍格式化」，或「任务做完就弹个桌面通知」——你去泡杯茶，它办完会叫你。本章机制里最能体现「自动化」的一环。

官方文档

自动化内置★★★★☆

Skills 技能 · 自定义 /命令

把你反复交代的一整套流程「打包」成一条命令。写一个 SKILL.md 文件放进 .claude/skills/，就多出一个 /技能名，需要时才加载、平时不占空间。场景：你常做「整理下载文件夹：按类型分类、删重复、生成清单」，做成一个 /整理下载，以后一句话搞定。它遵循开放的 Agent Skills 标准，能跨工具复用。

官方文档

扩展内置★★★☆☆

output styles 输出风格

换掉它「软件工程师」的默认人设，改变它「怎么说话、以什么格式回复」。内置 Explanatory（边做边讲解）、Learning（让你也动手参与）等。场景：你根本不写代码，只想让它当「写作助手」或「数据分析师」，自定义一个不带工程师包袱的风格，它的回复就更贴合你的用途。用 /config 里的 Output style 切换。

人设内置★★☆☆☆

MCP 接入

MCP（Model Context Protocol，模型上下文协议）是一个开放标准，让 Claude 连上外部工具和数据。装好对应的 MCP 服务器，它就能读你 Google Drive 里的文档、更新 Jira 工单、从 Slack 拉消息。场景：让它「照着云盘里那份设计稿的要求，把这份表格改好」，它能直接去读云端文件。第肆章会专门展开讲。

官方文档

连接内置★★★☆☆

git worktrees 并行开工

给同一个项目开好几个「平行副本」，让多个 Claude 会话同时干活、互不踩脚。场景：你想同时试三种不同的方案，各开一个 worktree 各跑一个 Claude，谁好用谁，互不干扰。偏技术，但对「想让 AI 同时多路推进」的重度用户是关键一招。/batch 命令也用它来批量改文件并各自发起 PR。

发现它跑偏了，按一下 Esc 立刻叫停当前动作，已经做的部分会保留，你可以马上纠正方向。双击 Esc：输入框有字时清空草稿，输入框为空时打开回溯菜单。场景：它正准备大改一堆文件、你突然想起漏交代了一个要求——Esc 打断，补一句，再让它继续。让你随时握着方向盘。

Skills 技能库精选

Skill（技能）是给 Claude 装的一本本「操作手册」：一个装着说明书、模板和小脚本的文件夹，平时不占地方，一旦你问的事正好对得上，Claude 就自动翻开照做。它让 Claude 从「聪明但外行」变成「聪明又懂你这行规矩」——会用你公司的模板做 PPT、按你的排版改 Word、照你定的流程写代码。本章先讲清 Skill 是什么、怎么装，再把官方与社区里真正好用的技能逐个成卡，非程序员也能照着上手。

Skills 到底是什么

一句话：Skill 是一个文件夹，里面有一份叫 SKILL.md 的说明书，告诉 Claude「遇到这类活儿，按这几步、用这些模板来做」。文件夹里还能放示例、字体、Word 模板、甚至一小段现成脚本。妙处在于「渐进式装载」——平时 Claude 只瞥一眼每个技能的一句话简介（几十个字的开销），只有你真要做那件事时，它才把整本手册读进来。用独立评论人 Simon Willison 的话说，这个机制简单到「可能比 MCP 还要重要」。

新手最容易把三个词搞混：Skill、MCP、子代理（subagent）。它们分工不同，配合起来才顺手。记住一句：Skill 教「怎么做」，MCP 接「外部的东西」，子代理派「另一个分身去干」。

能力解决什么打个比方
Skill 技能教 Claude 一套做事的方法与规矩（做 PPT、改 Word、按流程写代码）。纯说明书＋模板，不联网、不调外部服务。塞给它一本《本店操作手册》
MCP 服务器把 Claude 接到外部世界：数据库、Gmail、Notion、你公司的系统。是「连接线」，详见第肆章。给它接上电话和门禁卡
子代理 Subagent开一个独立上下文的「分身」去专门跑一件子任务，干完把结果带回来，不污染主对话。派一个助理去隔壁把活干完

怎么装、装在哪

技能对三种 Claude 都通用，装法各不同（以 2026 年官方文档为准）：

你用的是怎么装技能
Claude Code（命令行）官方技能市场一键装：先 /plugin marketplace add anthropics/skills 登记市场，再 /plugin install document-skills@anthropic-agent-skills 装文档四件套。自己的技能则放进两个文件夹：~/.claude/skills/（个人级，所有项目通用）或项目里的 .claude/skills/（项目级，跟着这个项目走）。文件夹名就是技能名。
Claude.ai 网页版 / App付费方案在「设置 → Capabilities（能力）」里开启技能，或直接上传自定义技能包。官方的文档技能默认已内置。
Claude API开发者按官方 Skills 指南把技能随请求带上，给自己的产品加能力。
新手须知：「plugin marketplace（插件市场）」听着吓人，其实就是一句命令登记一个「技能来源」，之后像手机应用商店那样挑着装。你完全不必懂 GitHub——把上面 Claude Code 那两行命令原样敲进去即可。想看官方怎么说，见 「什么是技能」帮助页与 Claude Code 技能文档。

MCP：让 Claude 连接万物

Claude 本身只会聊天——它看不到你的日历、读不到你家的传感器、也点不动网页上的按钮。MCP（Model Context Protocol，模型上下文协议）就是那根「插头」：一个开放标准，让 Claude 和外部工具用统一接口对接，就像 USB-C 之于所有充电器。插上一个 MCP 服务器，Claude 就多长一只手、一双眼。本章带你从「怎么插」到「插什么」，最后专门写透读者最想要的一节——让 Claude 看见你的手机和家。

MCP 是什么：给非程序员的一个比喻

把 Claude 想成一台很聪明、但没接任何外设的主机：会思考，却没有摄像头、没有硬盘、没有网线。MCP 服务器就是你往它身上插的一件件 USB 外设——插「文件读写」它就能翻你的文件夹，插「浏览器」它就能自己上网点按钮，插「智能家居」它就能读你家温度、开你家灯。协议由 Anthropic 于 2024 年 11 月提出，2025 年 OpenAI、Google 相继采用，2025 年底并入 Linux 基金会旗下，如今已是 AI 与工具对接的事实标准，服务器数量以千计。

你不需要懂技术。作为 Claude 用户，你只要知道三种「插外设」的方法，对应三个入口，各有官方文档。下面三张卡讲清各自适合谁、怎么开。

方法一 · claude.ai 的 Connectors（连接器）

最省心的一种，全程点鼠标、不碰命令行。在 claude.ai 或桌面版里打开「Customize（自定义）→ Connectors」，就能从官方连接器目录（2026 年中已有 400+ 个通过审核的集成，覆盖 Google Drive、Gmail、Notion、Slack、Stripe、Figma 等）一键开启；也能点「+ 添加自定义连接器」填一个远程 MCP 服务器网址。免费到企业版都能用（免费版限 1 个）。远程连接器由 Anthropic 云端代为连接，你的设备不用常开。不知道从哪开始就选它。

方法二 · Claude Code 的 claude mcp add

给用 Claude Code（第贰章那个命令行版）的人。装一个服务器就一句话：远程服务器用 claude mcp add --transport http 名字 网址，本地服务器用 claude mcp add 名字 -- 启动命令。装完 claude mcp list 看清单、在对话里敲 /mcp 查状态。配置按「作用域」分：只给当前项目、给你所有项目、或写进项目的 .mcp.json 供团队共享。功能最全，代价是要打几行字。

社区智慧与方法论

工具会更新，心法更耐用。这一章不讲按哪个键，讲的是「怎么想」——把 Anthropic 团队、Claude Code 的创造者、以及中英文社区里那些真正改变过人做法的话，挑成一面「金句墙」；再把大家反复验证、公认有效的几套工作流，写成你能照着跑的「配方卡」。所有金句都附了核对过的出处，你可以自己点进去看。

「金句墙」怎么读

下面每一条都经过一个标准筛选：它得是「让人改变做法的话」，而不是「正确的废话」。宁可少而真，也不要多而空。凡是英文原话，都给了中文转述加关键英文原文，方便你抓住那个词。它们分四组：模型分工与并行、上下文卫生、先规划后验证、心法与纪律。挑对你此刻最有用的那几条抄下来，就够回本。

一句话总纲：Claude 用得好不好，差距很少在「提示词写得多花哨」，而在你围着它搭起的那套章法——记忆文件、清理习惯、验证闭环、并行结构。这一章讲的就是那套章法。

一、模型分工与并行开工

读者点名想弄明白的那句「让贵模型做顶层设计、便宜模型执行」——它不是坊间偏方，而是 Anthropic 官方实测出来的最优结构，也是本章最值得先记住的一条。

二、上下文卫生：越干净越聪明

「上下文」＝这次对话里 Claude 记得的所有东西。它是 Claude Code 一切最佳实践的地基——几乎所有技巧，本质都是在替你打扫这块地方。

三、先规划、后动手、再自我验证

让 AI 直接开干最容易翻车的地方，是它会把「错的问题」解得很漂亮。这一组讲的是怎么先想清、再放手，以及怎么让它自己发现自己的错。

四、心法、纪律与边界感

最后一组关乎态度：高手和普通人的差距到底在哪、什么时候必须自己把关、以及「担责」这件事永远不能外包给 AI。

方法论配方：可跟做的工作流

金句是「为什么」，配方是「怎么做」。下面八张卡把社区公认有效的工作流拆成了几步走的清单——每张标了适用场景、具体步骤和出处，难度星越少越省心（★＝照着敲就行，★★★★★＝要花点功夫折腾）。不必全上，先挑一张最贴你当下的，跑通一次再加下一张。

「规划—执行—复查」三段式

最通用的一套骨架，适用于任何多文件、你没十足把握、或改动较大的任务。做法：①按 Shift+Tab 切到 Plan Mode（只读不改），让它先摸清现状；②让它写一份实施计划，你过目、要改就改；③退出 Plan Mode 放它动手，并要求它「对照计划做」；④完成后，另开一个「全新上下文」的子代理只看改动、做对抗式复查，报告缺口。把「想清楚」和「动手」分开，是不解错问题的关键。

灵感画廊：vibe coding 案例

这一章不讲工具，只看成品。「vibe coding（凭感觉编程）」是 2023 年由 Andrej Karpathy 起的名字——你用大白话描述想要什么，AI 把代码写出来，你几乎不看它到底怎么写的。下面是 2025 下半年到 2026 年，一群普通人（医生、修表匠、大学生、上班族）用 Claude 做出来的东西。每张卡都配了能看的原帖、视频或可玩链接；没公开代码的照收，标「仅演示·闭源」——那正是给你攒的灵感库。看完想动手，这章就成了。

怎么读这一章：「开源」= 代码公开在 GitHub 上，你能照抄、能改；「仅演示·闭源」= 只能看不能抄，但思路值得偷。星级是你想复刻的上手难度：★=有手就行，★★★★★=得折腾半天。★数量级（如 4.9k★）指该项目在 GitHub 的收藏数，是热度信号不是门槛。

一、新手词典：25 个绕不开的词

下面每张卡片都是「一句人话 + 和你的关系」。不用背，遇到不懂的词回来查即可。术语按「先会遇到的」大致排序，专有名词保留英文，方便你复制去搜索。

GitHub

是什么：全球最大的代码「云盘 + 协作广场」，程序员把项目放上去、公开或私有、互相协作。和你：本百科几乎每个项目的"主页"都在 GitHub。你只需会三件事——看 README、点 star 收藏、复制网址丢给 Claude Code。

仓库 Repository / repo

是什么：GitHub 上一个项目的完整文件夹，装着全部代码、说明和历史。一个 repo = 一个项目。和你：别人说"这个 repo 很好用"，就是"这个项目很好用"；网址长这样 github.com/作者/项目名。

Star（星标）

是什么：给仓库点的"收藏 + 赞"，星越多通常越受欢迎。~20k★ 读作"两万星"。和你：本百科用星数当热度信号（不是唯一标准）；你也能登录后给喜欢的项目点星，日后在自己主页一键找回。

Fork（分叉）

是什么：把别人的仓库整份复制到你自己名下，从此随便改，不影响原版。和你：想在某项目基础上改造、又怕弄坏原版时用它。日常评估项目，你多半用不到，认识即可。

Clone（克隆）

是什么：把云端仓库完整下载到你自己电脑上，变成一个本地文件夹。和你：要真正跑起某个项目，第一步就是 clone。好消息：你不用记命令，直接让 Claude Code "帮我 clone 这个链接到某文件夹"即可。

README

是什么：仓库的"首页说明书"，打开仓库自动显示，讲这是什么、怎么装、怎么用。和你：评估一个项目值不值得上手，先扫一眼 README。看不懂就整段复制给 Claude："用大白话讲这个项目能帮我做什么"。

Issue（问题单）

是什么：仓库自带的"公开留言板 / 报修单"，用来报 bug、提需求、问问题。和你：遇到项目跑不起来，先去 Issues 搜有没有人遇到同样的坑——常常已有现成答案。

Pull Request / PR（合并请求）

是什么："我改好了一段代码，请求把它合并进你的项目"的正式申请。开源协作的核心动作。和你：作为使用者你几乎不用发 PR；但看到项目"最近有活跃 PR"，说明它还在维护，是个好信号。

开源许可 License

是什么：作者写明"你能怎么用我的代码"的法律条款。常见的 MIT / Apache 最宽松，几乎随便用。和你：只要看到 MIT、Apache-2.0，个人自用尽管放心。拿不准就问 Claude："这个 License 允许我商用吗？"

Markdown

是什么：一种极简排版语法，用 # 表标题、* 表要点，纯文本就能写出漂亮文档。README 和 Claude 的回复都用它。和你：你在 Claude 里看到的加粗、列表、标题，背后就是 Markdown；想让 Claude 给你可复制的笔记，说"用 Markdown 输出"。

终端 / 命令行 Terminal

是什么：那个"黑底白字、只能打字"的窗口，用敲命令的方式指挥电脑，比点鼠标更快更强。和你：Claude Code 就住在终端里。你不必学命令——用中文说需求，让它替你敲。

CLI（命令行工具）

是什么：Command-Line Interface，"要在终端里用命令操作"的软件。Claude Code 本身就是一个 CLI。和你：看到某工具标注"CLI"，意味着它没有图形界面，靠打字使用——正好交给 Claude Code 代劳。

API key（接口密钥）

是什么：一串像密码的字符，程序凭它调用某个服务（如某个 AI 或天气接口）并计费。和你：把它当银行卡密码，绝不截图外发、不贴进公开仓库。给 Claude 配置时也要小心放在专门的密钥文件里。

Token（词元）

是什么：AI 处理文字的最小计量单位，约等于半个汉字或几个字母。模型按 token 计费、也按 token 算长度。和你："这段太长会爆 token"就是"太长了模型装不下"。日常记住：越长越贵、越慢即可。

上下文窗口 Context window

是什么：模型一次能"同时记住"的文字上限（如 20 万 token）。超过就会遗忘最早的内容。和你：聊太久 Claude"忘事"、答非所问，多半是窗口塞满了。开新对话、或让它先总结，就能缓解。

幻觉 Hallucination

是什么：AI 一本正经地编造不存在的事实、链接或函数。语气越自信越要警惕。和你：凡涉及数字、网址、报价，务必让它给来源、你再核实。本百科的"反幻觉链接纪律"就是为治它而生。

Prompt（提示词）

是什么：你发给 AI 的那段话/指令。写得越清楚、越有背景和示例，回复越好。俗称"咒语"。和你：用好 Claude 的关键 80% 在于会写 prompt。本章末尾直接送你两条可复制的现成咒语。

Agent（智能体）

是什么：能自己规划步骤、调用工具、连续行动来完成目标的 AI，而不只是一问一答。和你：Claude Code 就是个 agent：你说"把这项目跑起来"，它会自己看文件、装依赖、执行、报错再修——你只需拍板。

IDE（集成开发环境）

是什么：写代码的"超级编辑器"，集编辑、运行、调试于一体，最流行的是 VS Code。和你：Claude Code 有 VS Code 插件版，能在编辑器里直接改代码、看改动对比。非程序员用终端版也完全够。

JSON

是什么：一种用 { } 和"键: 值"表示数据的通用格式，人机都能读，配置文件和接口数据几乎都用它。和你：配置 Claude、MCP 时会碰到 .json 文件。改错一个逗号就会报错——交给 Claude 帮你改最稳。

localhost（本机地址）

是什么：代表"你自己这台电脑"的网址，常见 localhost:3000。项目跑起来后在这里预览。和你：Claude 说"打开 localhost:3000 看看"，就是让你在浏览器输入它，看本地跑起来的效果。只有你自己能访问。

部署 Deploy

是什么：把在你电脑上跑通的项目搬到公网服务器，让别人也能通过网址访问，即"上线"。和你：做完一个小网站想分享给朋友，就需要部署。可以直接让 Claude"帮我部署到 Vercel/Netlify 并给我网址"。

前端 / 后端 Frontend / Backend

是什么：前端 = 你看得见、能点的界面；后端 = 幕后处理数据和逻辑的服务器。和你：描述需求时分清这两半，Claude 更好动手："前端加个按钮"或"后端存一下这个数据"。

数据库 Database

是什么：专门有条理地存放数据的"电子仓库"，能快速增删改查，比 Excel 强大得多。和你：凡是要"记住用户、保存记录"的应用都离不开它。入门可让 Claude 用最简单的 SQLite 帮你起步。

正则表达式 Regex

是什么：一套"用符号描述文本规律"的迷你语言，专门做批量查找/替换/校验（如提取所有邮箱）。天书般难读。和你：永远不用手写。直接说"帮我写个正则，匹配所有手机号"，让 Claude 生成并解释即可。

依赖 Dependency

是什么：项目要用到的"现成零件包"（别人写好的库）。跑项目前得先"装依赖"，把这些零件下载齐。和你：常见报错就是缺依赖。不用管细节，说"帮我装好依赖再运行"，Claude 会自动搞定。

环境变量 Environment variable

是什么：存在系统或 .env 文件里的"外部配置值"，常用来放 API key 等不该写进代码的秘密。和你：项目要你"填 .env"，就是把密钥填进那个文件。让 Claude 告诉你每一项该填什么。

MCP（模型上下文协议）

是什么：Model Context Protocol，一套让 Claude 安全连接外部工具与数据（文件、数据库、网页、你的软件）的通用"插座标准"。和你：装上对应 MCP，Claude 就能读你的文件、查数据库、操作其他 App。详见第肆章。

Skill（技能）

是什么：打包好的"专长说明书 + 脚本"，Claude 需要时自动加载，瞬间获得某项专业能力（做 PPT、审文档、按某风格设计）。和你：装一个 Skill 就像给 Claude 请了位专科顾问。精选见第叁章。

Vibe coding（氛围编程）

是什么：不手写代码，只用自然语言描述想法、让 AI 生成并迭代的新式编程方式，2025 年爆火的词。和你：正是本百科为你准备的玩法——你出创意，Claude 出实现。第柒章满是灵感案例。

CLAUDE.md

是什么：放在项目根目录的一份"给 Claude 的常驻须知"，写清项目背景、规范、偏好，每次都会自动读取。和你：把你的口味写进它（如"回复用中文、别自作主张删文件"），Claude 就长期记住，不用每次重申。

Commit（提交）

是什么：用 Git 给项目当前状态存一个"存档点"，带一句说明，日后可随时回退。和你：把它当游戏存档。改动前让 Claude"先 commit 一下"，万一改坏了能一键还原，安心大胆试。

我是编程新手，请全程用大白话、中文回答，每一步做什么先告诉我。我想试用这个开源项目：<把 GitHub 链接粘这里>。请按顺序帮我：
1. 先打开它的 README，用三句话告诉我"这是什么、能帮我做什么、值不值得我这种小白上手"；
2. 评估它是否可信——看 star 数量级、最近有没有维护、开源许可是否宽松、有没有明显安全风险，给我一个"放心用 / 谨慎 / 别碰"的结论和理由；
3. 如果值得，把它 clone 到 桌面的 projects 文件夹（没有就新建），并告诉我它现在在哪；
